# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Contact Intelligence.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla CSS with CSS custom properties |
| Preset | not applicable |
| Component library | none |
| Icon library | Unicode emoji / inline SVG (existing pattern) |
| Font | DM Sans (Google Fonts CDN), DM Mono for monospace elements |

**Source:** Inherited from Phase 1 UI-SPEC. No changes to the design system foundation.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, staleness dot spacing, badge margin-top |

| md | 16px | Contact card padding, modal inner padding, form field spacing, AI content padding |

## Color

All values from existing `:root` CSS custom properties. No new colors introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--dark: #080C16` | Page background, modal overlay base |
| Secondary (30%) | `--surface: #141B2D` / `--card: #1A2236` | Contact cards (`--card`), modal background, contact detail expanded area |
| Accent (10%) | `--accent: #3B82F6` | Reserved list below |
| Destructive | `--red: #EF4444` | Delete contact button and confirmation only |

Accent (`#3B82F6`) reserved for:
- "Add Contact" primary CTA button background
- "Save Contact" primary CTA button background
- "Generate" AI button background
- Focused input border (existing pattern)
- Active tab bottom border (existing pattern)
- Contact card hover left-border highlight

Accent2 (`--accent2: #60A5FA`) reserved for:
- Active tab text (existing pattern)
- Contact detail section header labels
- Outreach log channel/outcome badge text

Status colors for influence badges:
- `--green: #22C55E` — Champion badge background tint and text
- `--gold: #F59E0B` — Evaluator badge background tint and text
- `--red: #EF4444` — Blocker badge background tint and text

Staleness indicator colors (per D-12):
- `--green: #22C55E` — Fresh: researched less than 30 days ago
- `--gold: #F59E0B` — Aging: researched 30-90 days ago
- `--red: #EF4444` — Stale: researched more than 90 days ago

**Influence badge color mapping:**

| Influence | Background | Border | Text Color |
|-----------|-----------|--------|------------|
| Champion | `rgba(34,197,94,0.1)` | `rgba(34,197,94,0.2)` | `var(--green)` |
| Evaluator | `rgba(245,158,11,0.1)` | `rgba(245,158,11,0.2)` | `var(--gold)` |
| Blocker | `rgba(239,68,68,0.1)` | `rgba(239,68,68,0.2)` | `var(--red)` |

**Staleness badge color mapping:**

| State | Background | Border | Text Color | Label |
|-------|-----------|--------|------------|-------|
| Fresh (<30d) | `rgba(34,197,94,0.1)` | `rgba(34,197,94,0.2)` | `var(--green)` | `FRESH` |
| Aging (30-90d) | `rgba(245,158,11,0.1)` | `rgba(245,158,11,0.2)` | `var(--gold)` | `AGING` |
| Stale (>90d) | `rgba(239,68,68,0.1)` | `rgba(239,68,68,0.2)` | `var(--red)` | `STALE` |

**Focal point:** The contact name at 13px / 500 weight is the primary visual anchor within each card. Influence badges provide the secondary visual signal via color.

## Component Patterns

### Contacts Tab (new in Phase 2)

**Tab placement:** "Contacts" tab is inserted as the second tab after "Overview" and before all existing tabs (Tech Stack, Talent Signals, Executives, Ask AI). This places people-data immediately after the account summary.

Tab button follows existing `.acct-tab` pattern. No new CSS needed for the tab itself.

### Contact Card (new in Phase 2)

Contact cards follow the existing `.exec-card` pattern as a starting point but are extended for click-to-expand behavior.

**Structure:**
```
.contact-grid    -- display: grid, grid-template-columns: repeat(auto-fill, minmax(260px, 1fr)), gap: 16px
  .contact-card  -- background: var(--card), border: 1px solid var(--border),
                    border-radius: 8px, padding: 16px, cursor: pointer,
                    transition: border-color 0.12s, border-left: 3px solid transparent
    .contact-card-hdr  -- display: flex, align-items: center, gap: 8px
      .contact-av      -- width: 32px, height: 32px, border-radius: 50%,
                          background: linear-gradient(135deg, var(--accent), var(--purple)),
                          display: flex, align-items: center, justify-content: center,
                          font-size: 11px, font-weight: 500, color: white (initials)
      .contact-info    -- flex: 1
        .contact-name  -- font-size: 13px, font-weight: 500
        .contact-title -- font-size: 11px, color: var(--muted), margin-top: 4px, line-height: 1.4
      .contact-badges  -- display: flex, gap: 4px, flex-shrink: 0
        .influence-badge -- (see Influence Badge below)
        .stale-badge     -- (see Staleness Badge below)
```

**States:**

| State | Behavior |
|-------|----------|
| Default | Card at rest with muted border |
| Hover | `border-color: rgba(59,130,246,0.3)` (matches `.exec-card:hover`) |
| Expanded | `border-left-color: var(--accent)`, detail view visible below card header |
| Empty group | Group header visible, "No [influence] contacts yet" in muted text |

### Influence Group Header (new in Phase 2)

Contacts are grouped by influence level. Each group has a header.

```css
.contact-group { margin-bottom: 24px; }
.contact-group-hdr {
  font-size: 11px;
  font-weight: 500;
  text-transform: uppercase;
  letter-spacing: 0.1em;
  color: var(--muted);
  margin-bottom: 8px;
  padding-bottom: 8px;
  border-bottom: 1px solid var(--border);
}
```

Group order: Champions first, then Evaluators, then Blockers. Groups with zero contacts are hidden entirely.

### Contact Detail View (new in Phase 2)

**Decision:** Inline expansion that pushes content down (not overlay). Click a contact card to expand; click again or click another card to collapse. Only one card expanded at a time.

The detail view renders inside the `.contact-card` element, below the `.contact-card-hdr`.

```
.contact-detail   -- padding: 16px 0 0, margin-top: 8px,
                     border-top: 1px solid var(--border), display: none
.contact-detail.show -- display: block

  .contact-section        -- margin-bottom: 16px
    .contact-section-hdr  -- font-size: 11px, font-weight: 500, text-transform: uppercase,
                             letter-spacing: 0.08em, color: var(--accent2), margin-bottom: 8px
    .contact-section-body -- font-size: 13px, line-height: 1.6, color: var(--text)

  .contact-fields         -- display: grid, grid-template-columns: 1fr 1fr, gap: 8px 16px, margin-bottom: 16px
    .contact-field-label  -- font-size: 11px, color: var(--muted)
    .contact-field-value  -- font-size: 13px, color: var(--text)
```

**Detail sections (in order):**
1. **Contact info** — Email, LinkedIn, Phone (grid layout, 2-column)
2. **AI Outreach Rationale** — Section with body text or "Generate" button if empty
3. **Warm Path** — Section with body text or "Generate" button if empty
4. **Outreach History** — Reverse-chronological list with inline add form
5. **Actions** — Edit Contact, Delete Contact, Refresh AI buttons

### AI Generate Button (new in Phase 2)

Used when `ai_rationale` or `warm_path` is empty. Also used for refresh.

```css
.contact-generate-btn {
  background: var(--accent);
  border: none;
  border-radius: 8px;
  padding: 8px 16px;
  font-size: 13px;
  font-family: var(--font);
  color: #fff;
  cursor: pointer;
  transition: background 0.12s;

.contact-generate-btn:hover { background: #2563EB; }
```

When generating: button text changes to "Generating..." with disabled state. On success: rationale/warm path text replaces the button. On error: red error message below the button.

### Outreach Inline Form (new in Phase 2)

Per D-09, outreach attempts are logged via inline form in the detail view (not a modal).

```
.outreach-form        -- display: flex, flex-direction: column, gap: 8px,
                         padding: 16px, background: var(--surface),
                         border: 1px solid var(--border), border-radius: 8px, margin-top: 8px
  .outreach-form-row  -- display: flex, gap: 8px
    select.form-input -- (channel dropdown, same .form-input style)
    select.form-input -- (outcome dropdown, same .form-input style)
    input.form-input  -- (date picker, type="date", same .form-input style)
  textarea.form-input -- (notes, 2 rows, optional)

    button.btn.btn-secondary -- "Discard"
    button.btn.btn-primary   -- "Log Outreach"
```

The form is hidden by default and revealed by a "+ Log Outreach" link button.

### Add/Edit Contact Modal (reuse Phase 1)

Reuses the existing `.modal-overlay`, `.modal`, `.modal-hdr`, `.modal-body`, `.modal-ftr` pattern from Phase 1 (per D-04). Max-width remains 480px.

**Form fields (in order):**

| Field | Type | Required | Placeholder |
|-------|------|----------|------------|
| Name | text input | Yes | `Full name` |
| Title | text input | Yes | `Job title (e.g. VP Digital Engineering)` |
| Influence | select dropdown | Yes | Options: Champion, Evaluator, Blocker |
| Email | email input | No | `Email address` |
| LinkedIn | text input | No | `LinkedIn profile URL` |
| Phone | tel input | No | `Phone number` |

Name, title, and influence are required. Email, LinkedIn, and phone are optional. Required fields show `var(--red)` border on submit if empty. The `.form-error` class from Phase 1 displays validation messages.

### Delete Contact Confirmation (reuse Phase 1)

Same confirmation modal pattern as Phase 1 account removal. Contact soft-deletes per D-18.

## Interaction States

### Contacts Tab

| State | Behavior |
|-------|----------|
| Default (has contacts) | Contact grid with cards grouped by influence level. Count shows in `.contacts-count`. |
| Default (no contacts) | Empty state centered in tab pane with heading, body, and "Add Contact" CTA below. |
| Loading | Cards render from API response on tab activation. No skeleton -- fast enough for local SQLite. |

### Contact Card

| State | Behavior |
|-------|----------|
| Default | Card at rest, border `var(--border)`, border-left 3px transparent |
| Hover | Border color shifts to `rgba(59,130,246,0.3)` |
| Expanded | Border-left becomes `var(--accent)`, `.contact-detail.show` slides open below header |
| Collapse | Click same card again or expand different card -- collapses with `display: none` |

### AI Generation

| State | Behavior |
|-------|----------|
| Idle (no content) | "Generate Rationale" / "Find Warm Path" button visible |
| Generating | Button text changes to "Generating...", disabled, `opacity: 0.6` |
| Success | Button replaced by AI content block (`.ai-content-block`) with "Refresh Rationale" or "Refresh Warm Path" link |
| Error | Red error message at 11px below button: "Couldn't generate. Try again in a moment." |
| Has content | AI content block displayed. Contextual refresh link at bottom re-triggers generation. |

### Outreach Log Form

| State | Behavior |
|-------|----------|
| Hidden | Form not visible. "+ Log Outreach" link visible below outreach list. |
| Visible | Click "+ Log Outreach" reveals inline form. Focus moves to date field. |
| Submitting | "Log Outreach" button shows "Saving..." text, disabled. |
| Success | Form hides, new entry prepended to outreach list (optimistic UI from API response). |
| Error | Red error message below form. |
| Discard | "Discard" button hides the form without saving. |

### Add/Edit Contact Modal

| State | Behavior |
|-------|----------|
| Closed | `display: none` on `.modal-overlay` |
| Open (add) | Empty form, title "Add Contact" |
| Open (edit) | Pre-filled form, title "Edit Contact" |
| Validation error | Red border on empty required fields, error message below each |
| Submitting | "Save Contact" / "Save Changes" shows "Saving...", disabled |
| Success | Modal closes, contact grid re-renders with new/updated card |
| Error | Error message below form in `var(--red)` at 11px |

### Delete Contact Confirmation

| State | Behavior |
|-------|----------|
| Open | Modal with contact name in body text, "Delete Contact" destructive CTA |
| Confirming | "Delete Contact" shows "Deleting...", disabled |
| Success | Modal closes, card removed from grid, contact count updates |

### Staleness Badge

| State | Condition | Behavior |
|-------|-----------|----------|
| Fresh | `researched_at` < 30 days ago | Green badge with "FRESH" label |
| Aging | `researched_at` 30-90 days ago | Gold badge with "AGING" label |
| Stale | `researched_at` > 90 days ago | Red badge with "STALE" label |
| Never researched | `researched_at` is null | Red badge with "STALE" label (same as >90d) |
