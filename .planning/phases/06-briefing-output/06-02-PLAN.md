  <action>
Add a `@media print` block at the END of the `<style>` section in index.html (after all existing CSS rules, just before the closing `</style>` tag). Every color override MUST use `!important` to beat dark theme specificity (per D-14, Pitfall 1).

```css
@media print {
  /* Hide all app chrome */
  .topnav,
  .sidebar,
  .acct-header,
  .acct-tabs,
  .briefing-actions,
  .stale-badge,
  .acct-actions { display: none !important; }

  /* Reset dark theme to white page */
  body, html { background: #fff !important; color: #000 !important; }
  .main-content { margin: 0 !important; padding: 0 !important; background: #fff !important; }
  .acct-content { border: none !important; background: #fff !important; padding: 0 !important; }

  /* Hide all tab panes except the active one (the briefing) */
  .tab-pane { display: none !important; }
  .tab-pane.active { display: block !important; }

  /* Account panel: remove dark styling */
  .account-panel { background: #fff !important; border: none !important; }

  /* Briefing card: clean print appearance */
  .briefing-card { box-shadow: none !important; border: none !important; background: #fff !important; color: #000 !important; padding: 0 !important; }
  .briefing-content { font-size: 12px !important; line-height: 1.6 !important; color: #000 !important; }
  .briefing-meta { color: #666 !important; font-size: 11px !important; }
  .briefing-section-header { color: #000 !important; border-bottom: 1px solid #ccc !important; padding-bottom: 4px !important; margin-top: 16px !important; page-break-inside: avoid !important; }

  /* Page settings */
  @page { margin: 20mm; size: A4 portrait; }
}
```

Per D-11: Hides navigation, sidebar, tabs, and non-briefing elements.
Per D-12: Formats briefing as clean single-page document with margins and typography.
Per D-13: Print button (from Plan 01) already triggers `window.print()`.
Per D-14: DM Sans (already loaded), black text on white background, no dark theme.

Critical ordering: `.tab-pane { display: none !important; }` MUST come BEFORE `.tab-pane.active { display: block !important; }` so the active briefing pane remains visible (per Pitfall 2).

The `page-break-inside: avoid` on `.briefing-section-header` prevents section headers from being orphaned at page breaks (per RESEARCH.md Open Question 2).
