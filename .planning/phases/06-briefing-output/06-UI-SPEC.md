## Interaction States

### Briefing Tab First Open (no cached briefing)

1. `showTab()` fires — sets `.active` on tab button and pane
2. `loadBriefingTab()` initializes panel HTML with `.strategy-loading` ("Loading briefing...")
3. `loadBriefing()` fetches GET `/api/accounts/:id/briefing`
4. Server returns 404 — auto-generate triggers immediately
5. Loading copy updates to "Generating briefing..."
6. POST `/api/accounts/:id/briefing` fires
7. On success: `renderBriefing()` replaces loading state with `.briefing-card` content; `.briefing-actions` bar appears (display: flex)
8. On failure: `.strategy-empty` with "Failed to generate briefing. [Try again]" link

### Briefing Tab Subsequent Open (cached briefing exists)

1. `showTab()` fires — `dataset.loaded` guard prevents re-fetch
2. Cached content already rendered — no loading state, no API call
3. Both action buttons visible: "Regenerate Briefing" (default style) + "Print / Save as PDF" (default style)

### Regenerate Briefing

1. User clicks "Regenerate Briefing"
2. No confirmation dialog (no manual edits to protect)
3. Actions bar hides (display: none)
4. Loading state: "Regenerating briefing..."
5. POST `/api/accounts/:id/briefing` fires
6. On success: fresh briefing renders; actions bar reappears
7. On failure: `.strategy-empty` error with "Try again" link

### Print / Save as PDF

1. User must be on Briefing tab (button only appears on Briefing tab)
2. Click triggers `window.print()`
3. Browser opens native print dialog
4. `@media print` CSS hides all app chrome; briefing content fills page
5. User selects "Save as PDF" from print destinations

## Print Layout Contract

`@media print` CSS block — all rules require `!important` to override dark theme variables.

**Elements to hide:**
- `.topnav` — navigation bar
- `.sidebar` — account list sidebar
- `.acct-header` — account KPI header
- `.acct-tabs` — tab navigation row
- `.briefing-actions` — Generate/Regenerate/Print buttons
- `.stale-badge` — staleness indicators
- `.acct-actions` — any account-level action buttons

**Tab pane handling:**
```
.tab-pane { display: none !important; }
.tab-pane.active { display: block !important; }
```

**Color reset:**
- `body, html`: `background: #fff !important; color: #000 !important`
- `.main-content`: `margin: 0 !important; padding: 0 !important; background: #fff !important`
- `.acct-content`: `border: none !important; background: #fff !important`
- `.briefing-card`: `box-shadow: none !important; border: none !important; background: #fff !important; color: #000 !important; padding: 0 !important`
- `.briefing-content`: `font-size: 12px !important; line-height: 1.6 !important`
- `.briefing-meta`: `color: #666 !important; font-size: 11px !important`

**Page settings:**

**Page break handling:**

## AI Content Rendering Contract

The `briefingTextToHtml(text)` function converts stored plain text to display HTML:

1. Split on `\n`
2. Run `escapeHtml(line)` on EVERY line before any other processing (XSS safety)
3. If escaped line starts with `## `: wrap in `<h3 class="briefing-section-header">`, strip the `## ` prefix
4. Empty lines: render as `&nbsp;`
5. All other lines: output as-is, join with `<br>`

Security invariant: `escapeHtml()` MUST run before heading detection. The check `if (escaped.indexOf('## ') === 0)` runs on the already-escaped string — safe because `escapeHtml()` does not transform `## `.

Code fence strip: If AI response contains ` ``` ` characters, strip with:
```javascript
text = text.replace(/```(?:markdown)?\s*/g, '').replace(/```\s*$/g, '').trim();
```
Applied server-side before storing to DB, same pattern as intelligence refresh endpoint.
