**JavaScript functions** (add after the activity-related functions):

1. `loadStrategyTab(accountId)` -- Renders the Strategy tab skeleton HTML into the panel:
   - Strategy summary card section (with id `${accountId}-strategy-card`)
   - Divider
   - Private Intel section title "Private Intel"
   - Intel form: textarea + "Add Note" button per D-03
   - Intel notes list container (with id `${accountId}-intel-list`)
   - Placeholder for triggers section (id `${accountId}-triggers-section`, empty -- Plan 03 fills this)
   - Then calls `loadStrategy(accountId)` and `loadIntel(accountId)`

2. `loadStrategy(accountId)` -- Fetches GET /api/accounts/:id/strategy:
   - If response has content: call renderStrategy(accountId, data)
   - If content is null (no strategy yet): auto-generate per D-10 -- check `dataset.generating` flag to prevent race condition (per RESEARCH.md pitfall 4), set flag, call generateStrategy(accountId)
   - Show loading state during auto-generation: "Generating strategy..."

3. `renderStrategy(accountId, strategy)` -- Renders strategy summary card per D-06:
   - Card header: "Strategy Summary" title + badges
   - Badges: "AI-Generated" badge (always), "Edited" badge if is_edited === 1 per D-13
   - Show generated_at date, and edited_at if present
   - Content div showing strategy.content with escapeHtml() and newlines converted to `<br>` AFTER escaping
   - Action buttons: "Edit" and "Regenerate Strategy" per D-07, D-11

4. `generateStrategy(accountId)` -- POST /api/accounts/:id/strategy:
   - Show loading spinner in strategy card area
   - On success: renderStrategy(accountId, data)
   - On error: show error message, re-enable regenerate button
   - Clear `dataset.generating` flag on completion

5. `editStrategy(accountId)` -- Per D-11, D-12: enter edit mode
   - Replace strategy content div with textarea pre-filled with current strategy text (unescaped)
   - Show Save + Cancel buttons, hide Edit + Regenerate
   - Store original content for Cancel revert

6. `saveStrategyEdit(accountId)` -- PUT /api/accounts/:id/strategy:
   - Get textarea value, validate non-empty
   - On success: renderStrategy(accountId, updatedData)
   - Per D-12: Save persists, shows updated badge

7. `cancelStrategyEdit(accountId)` -- Per D-12: revert to display mode with original content

8. `regenerateStrategy(accountId)` -- Per D-14: checks if current strategy is_edited
   - If is_edited: show confirm("This will replace your edits. Continue?")
   - If confirmed (or not edited): call generateStrategy(accountId)

9. `submitIntel(accountId)` -- Per D-03, D-05: POST /api/accounts/:id/intel
   - Get textarea value, validate non-empty
   - Disable button, show "Saving..."
   - On success: clear textarea, prepend new note to list (reverse-chronological per D-02)
   - On error: re-enable button, show error

10. `loadIntel(accountId)` -- GET /api/accounts/:id/intel:
    - Render notes into intel-list container
    - Each note: intel-note card with "Internal" label (intel-label), timestamp (intel-date), content (intel-text) per D-02
    - Use escapeHtml() on all content
    - Use formatActivityDate() or similar for timestamps
    - If no notes: show "No private intel notes yet."
