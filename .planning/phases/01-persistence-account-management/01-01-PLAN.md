**Route: GET /api/accounts**
```javascript
if (req.method === 'GET' && parsed.pathname === '/api/accounts') {
  const accounts = db.getAccounts();
  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify(accounts));

```

**Route: GET /api/accounts/:id**
Parse the id from pathname: `const match = parsed.pathname.match(/^\/api\/accounts\/([a-z0-9-]+)$/);`
```javascript
if (req.method === 'GET' && match) {
  const account = db.getAccount(match[1]);
  if (!account) {
    res.writeHead(404, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ error: 'Account not found' }));
    return;
  }
  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify(account));
  return;
}
```

**Route: POST /api/accounts**
Collect body, JSON.parse, validate `name` field exists and is non-empty string. Call `db.createAccount(body)`. Return 201 with the new account object.
```javascript
if (req.method === 'POST' && parsed.pathname === '/api/accounts') {
  // collect body, parse JSON
  // validate: if (!body.name || typeof body.name !== 'string' || !body.name.trim())
  //   return 400 { error: 'Name is required' }
  const account = db.createAccount(body);
  res.writeHead(201, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify(account));
  return;
}
```

**Route: PUT /api/accounts/:id**
Collect body, JSON.parse, call `db.updateAccount(match[1], body)`. Return 200 with updated account. Return 404 if account not found.

**Route: DELETE /api/accounts/:id**
Call `db.deleteAccount(match[1])`. Return 200 with `{ success: true }`. This is a soft delete per D-06.

**Route: POST /api/accounts/:id/restore**
Call `db.restoreAccount(match[1])`. Return 200 with restored account.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Server API | All CRUD requests cross from untrusted client to server |
| Server -> SQLite | User-supplied data written to database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Injection | POST/PUT /api/accounts | mitigate | Use parameterized prepared statements in all db.js queries — never concatenate user input into SQL strings |
| T-01-02 | Tampering | DELETE /api/accounts/:id | mitigate | Validate id parameter matches /^[a-z0-9-]+$/ regex before passing to DB query |
| T-01-03 | Info Disclosure | GET /api/accounts | accept | Account data is intended to be visible to all authenticated users; auth cookie gates access |
| T-01-04 | Spoofing | All API routes | mitigate | Existing isAuthenticated() cookie check applied before all new routes (routes placed after auth guard) |
| T-01-05 | Denial of Service | POST /api/accounts | mitigate | Request body size implicitly limited by Node.js default (~1MB); account context field is large text but bounded by practical use |
| T-01-06 | Elevation | /api/accounts/:id/restore | accept | No role-based access — all authenticated users have same permissions (per project constraint: shared password auth) |
</threat_model>
