## Color

All values from existing `:root` CSS custom properties. No new colors are introduced in Phase 1.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--dark: #080C16` | Page background, modal overlay base |
| Secondary (30%) | `--surface: #141B2D` / `--card: #1A2236` | Modal background (`--card`), sidebar (`--mid: #0F1525`), input backgrounds |
| Accent (10%) | `--accent: #3B82F6` | Reserved list below |
| Destructive | `--red: #EF4444` | Remove account button, confirmation destructive state only |

Accent (`#3B82F6`) reserved for:
- "Add Account" primary CTA button background
- "Save Changes" primary CTA button background
- Active sidebar item left border (existing)
- Focused input border (existing)
- Active tab bottom border (existing)

Accent2 (`--accent2: #60A5FA`) reserved for:
- Active sidebar item text (existing)
- Active tab text (existing)
- Monospaced metadata labels (existing)

Status colors:
- `--green: #22C55E` — AI chat live indicator dot (existing), future success feedback
- `--gold: #F59E0B` — Warning states (reserved, not used in Phase 1 UI)
- `--red: #EF4444` — Remove account button and destructive confirmation only

Muted text (`--muted: #7C8DB5`) for: all secondary labels, placeholder text, helper text, sidebar section headers, revenue figures.

**Focal point:** The active account name in `.acct-header` at 28px / 400 weight on `--dark` background is the primary visual anchor. All other elements defer to it in hierarchy.

## Component Patterns

### Modal (new in Phase 1)

No modal pattern exists in the current codebase. Establish the pattern here for use across all phases.

**Structure:**
```
.modal-overlay   — fixed, full-viewport, rgba(8,12,22,0.85) backdrop, z-index: 1000
  .modal         — background: var(--card), border: 1px solid var(--border2),
                   border-radius: 10px, padding: 24px, max-width: 480px, width: calc(100% - 32px)
    .modal-hdr   — flex row, border-bottom: 1px solid var(--border), padding-bottom: 16px, margin-bottom: 20px
    .modal-body  — form fields stacked with 16px gap
    .modal-ftr   — flex row, justify-content: flex-end, gap: 8px, padding-top: 20px,
                   border-top: 1px solid var(--border), margin-top: 20px
```

**Behavior:**
- Opens via "Add Account" button click or edit icon click on account header
- Closes via Discard Changes button, X icon, or clicking the overlay (not keyboard Escape — not in scope, inline onclick pattern)
- Overlay click calls `closeModal()` via `onclick` on `.modal-overlay`
- Stop propagation on `.modal` via `onclick="event.stopPropagation()"`

### Form Input (new in Phase 1)

Match the existing `.ai-inp` pattern:

```css
background: var(--card);
border: 1px solid var(--border);
border-radius: 6px;
padding: 8px 12px;
font-size: 13px;
color: var(--text);
font-family: var(--font);
outline: none;
width: 100%;
```

Focus state: `border-color: var(--accent);`
Placeholder: `color: var(--muted);`

### Button Variants (new in Phase 1)

Three variants needed. Extend `.ai-send` pattern for primary; define secondary and destructive new.

| Variant | Background | Border | Text color | Use |
|---------|-----------|--------|------------|-----|
| Primary | `var(--accent)` | none | `#fff` | Save, Add Account |
| Secondary | `transparent` | `var(--border2)` | `var(--muted)` | Discard Changes |
| Destructive | `transparent` | `rgba(239,68,68,0.3)` | `var(--red)` | Remove Account |

All buttons: `border-radius: 6px`, `padding: 7px 16px`, `font-size: 13px`, `font-family: var(--font)`, `cursor: pointer`.
Hover states: Primary darkens to `#2563EB`; Secondary background becomes `rgba(255,255,255,0.04)`; Destructive background becomes `rgba(239,68,68,0.08)`.

### Add Account Trigger (new in Phase 1)

Placement: Bottom of sidebar, below the last sidebar section. Always visible, sticky.

```
.sidebar-add    — display: flex, align-items: center, gap: 8px, padding: 8px 16px,
                  margin-top: auto, border-top: 1px solid var(--border),
                  font-size: 13px, color: var(--muted), cursor: pointer, background: none
.sidebar-add:hover — color: var(--text), background: rgba(255,255,255,0.03)
```

Label: `+ Add Account`

### Edit / Delete Controls (new in Phase 1)

Placement: Revealed on `.acct-header` hover via sibling CSS or JS class toggle. Right-aligned, small icon buttons.

```
.acct-actions   — flex row, gap: 8px, align-items: center, margin-left: auto
.acct-action-btn — background: none, border: 1px solid var(--border), border-radius: 5px,
                   padding: 4px 8px, font-size: 11px, color: var(--muted), cursor: pointer
.acct-action-btn:hover — border-color: var(--border2), color: var(--text)
.acct-action-btn.del:hover — border-color: rgba(239,68,68,0.3), color: var(--red)
```

Labels: `Edit Account` / `Remove Account`

## Interaction States

### Account Panel Header

The `.acct-header` displays KPIs and account identity. In Phase 1, it gains Edit and Remove controls.

| State | Behavior |
|-------|---------|
| Default | Header shows account name, sector tag, KPI row. Edit/Remove Account buttons visible at small opacity (0.4) or hidden until hover. |
| Hover | Edit and Remove Account buttons appear at full opacity. |
| Edit click | Opens pre-filled modal with current account data. |
| Remove Account click | Opens a secondary confirmation modal (not inline — separate modal to prevent accidental deletion). |

### Modal

| State | Behavior |
|-------|---------|
| Closed | `display: none` on `.modal-overlay` |
| Open | `display: flex` with `align-items: center`, `justify-content: center` |
| Submitting | Primary button shows `Saving...` text, disabled state (`opacity: 0.6`, `pointer-events: none`) |
| Error | Error message appears below form fields in `var(--red)` at 11px |
| Success | Modal closes, sidebar updates immediately (optimistic UI from API response) |

### Sidebar Account Item

| State | Behavior |
|-------|---------|
| Default | `.sidebar-item` — muted text, no border |
| Hover | `rgba(255,255,255,0.03)` background, text color `var(--text)` (existing) |
| Active | `rgba(59,130,246,0.08)` background, `var(--accent2)` text, 2px left border `var(--accent)` (existing) |
| New account added | Appended to correct sector group, auto-selected and scrolled into view |

### Chat Persistence (PERS-04)

No new visual pattern. Existing chat UI is unchanged. The only behavioral change: on page load, chat history is restored from the server (DB) rather than starting empty. Restored messages render identically to live messages using the existing `.msg` / `.msg-bub` pattern. No "restored" label or visual indicator.
